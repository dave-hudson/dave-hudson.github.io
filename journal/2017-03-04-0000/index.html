c8: More incremental improvements

2017-03-04 00:00 - 11 min read

The last few days have seen some steady changes:

  • Implementations have become more regular.

  • Divide and modulus operations have been broken into pieces and made more efficient.

  • Comments have been improved.

  • Variable names have ben made more regular.

  • More logic has been moved into the digit-array layer, so the digit-array behaviour is no longer visible in the public headers.

None of this is particular radical stuff, but it makes the code easier to understand, and easier to improve in the future. It has a nice impact on the performance right now, though, as divide-heavy code (which includes all rationals) is anything from 10% to 20% faster:
